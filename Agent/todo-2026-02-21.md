### Known Edge Case — Backup Restores
Restoring a database backup will resurface soft-deleted accounts and posts. The cron will re-process them on its next run if they are past the grace period. CDN images deleted before the restore are permanently gone — restored DB records will reference missing files. **Rule:** database backups must never be older than the grace period (7 or 30 days, whichever applies). Restoring an older backup may require manual cleanup. This is an accepted operational risk — no code solution exists for it.

| # | Table / Resource | Action |

| 1 | Email 711 | Send BEFORE touching any data |
| 2 | Bunny CDN — avatar | DELETE file using `members.avatar_file` |
| 3 | Bunny CDN — all post images | DELETE all files from `post_media.file_url` across all member's posts (including soft-deleted rows) |

| 6 | `member_settings` | Hard DELETE WHERE `member_id` = this member |
| 7 | `posts` | Hard DELETE all rows WHERE `member_id` = this member — cascades handle all children (same as post deletion × all posts) |
| 8 | `members` / `admins` row | Wipe personal columns only — do NOT delete the row (see below) |
| 9 | `transactions` | Leave untouched — financial records |

**Members row — tombstone approach:** Do not hard DELETE the row. Instead wipe all personal columns (`username`, `username_key`, `password_hash`, `avatar_file`, `favorites`, `recent`, `country`, `preferred_currency`, `backup_json`, `filters_json`, `filters_hash`, `reminder_emails` etc.) and leave only: `id`, `account_email`, `deleted_at`, `hidden = 1`. This serves as a legal record (dispute handling, fraud prevention) and frees the email address for re-registration. Any future registration attempt with the same email will see the tombstone and allow a fresh account.

**Registration code must be updated:** The duplicate email check in `add-member.php` currently blocks registration if the email already exists in the `members` table. It must be updated to exclude rows where `deleted_at IS NOT NULL` — so a previously deleted email address can be used to create a new account.

### How `send-email.php` Works
- Called via the gateway like all other connectors
- POST params: `message_key`, `to_email`, `to_name`, `member_id`, plus any placeholder values declared in the template
- Reads template from `admin_messages` by `message_key`
- Substitutes declared `{placeholders}` only
- Fetches `support_email`, `website_name`, `email_logo`, `folder_system_images` from `admin_settings` at runtime — nothing hardcoded
- Injects logo header dynamically into email body
- Sends via PHPMailer (SSL, port 465)
- Logs result to `emails_sent` table regardless of success or failure

### Logo Injection
The logo is NOT stored in the email templates. It is injected by `send-email.php` at send time by inserting a header div after the opening wrapper div. The preview page (`email-preview.php`) does the same. To change the email logo, update the `email_logo` setting in the admin panel Settings tab.
