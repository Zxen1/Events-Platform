### 1. Subcategory Merge Threshold (Filter Panel)

Built a system where subcategories with low worldwide post counts are visually merged into a catch-all subcategory in the filter panel. Posts retain their real subcategory in the database and on postcards.

**Database:**
- Added `filter_rules` JSON column to `categories` table (after `hidden`)
- What's On (category ID 1) has: `{"catchall_key":"other-events","catchall_threshold":10,"comment":"..."}`
- Categories with NULL `filter_rules` are unaffected

**Server-side (`get-form.php`):**
- `fetchCategories()` now selects and passes through `filter_rules` (bug fix: the column was being selected but dropped during row reconstruction)
- `buildFormData()` JSON-decodes `filter_rules` into `categoriesMap`
- Added worldwide subcategory counts query (`SELECT subcategory_key, COUNT(*) FROM posts GROUP BY subcategory_key`) â€” ships as `formData.worldwide_counts`

**Client-side (`filter.js`):**
- Module-level `mergedSubs` map (subcategory_key â†’ catchall key)
- `initCategoryFilter()` builds merge map from `filter_rules` + worldwide counts; hides merged rows with `display:none` and `data-merged-into`
- `applyFacetCounts()` rolls merged sub counts into catchall via `adjustedMap`
- Catchall toggle syncs hidden sub toggles
- `syncMergedToggles()` runs after state restore for consistency
- Solo mode: soloing the catchall also includes all merged sub keys in the server request
- The catchall subcategory itself is always exempt from merging (never hidden)
